Урок 3.8 - Практика 2 - пагинация и табы

Асинк Табы

Суть задачи: написать скрипт (табы) который принимает в себя ссылки и показывает скачанную по ссылкам информацию в контенте табов. Информация должна скачиваться не сразу, а по клику на открытый таб. Если таб уже был открыт ранее, то при повторном открытии информация скачиваться не должна.

Откуда брать инфу

Самостоятельно изучить ресурс

Разобраться, какую информацию он предоставляет и как её скачивать на сайт (куда делать запрос)

Как сделать кнопки

Функция, которая создаёт табы, одним из аргументов должна принимать в себя список ссылок, которые и будут являться контентом для кнопок.

Имена кнопок можно сделать (два варианта для примера)

const linksObj = { users: 'url1', posts: 'url2', photos: 'url3', "так тоже должно работать": "url4" }

const linksArr = [ { name: "Пользователи", link: 'url1', }, { name: "Фотки", link: 'url2', }, { name: "Статьи", link: 'url3', }, ]

Как создавать контент внутри

Для создания контента можно придумать свой уникальный способ или воспользоваться одним из трёх стандартных:

  • Создавать через document.createElement
  • Создавать через шаблонные строки и innerHTML
  • Создавать через хтмл-шаблоны template и их клонирование через documentFragment.clone(true)
  • Использовать шаблонизатор (напр.handlebars)

Пагинация

Пагинация - данный скрипт должен получать список данных на вход и выдавать "групированный" список данных, в котором данные разбиты на группы (страницы). Размер группы определить самостоятельно. От размера группы будет зависеть конечное кличество страниц. Дизайн - взять любой.

<< < 16 17 [18] 19 20 > >>

Есть массив данных (большой). Мы не хотим показывать все данные сразу, мы хотим разбивать массив на страницы по N элементов на страницу (например по 10 или по 20).

Дополнительно - получать не все данные сразу, а получать только 1-3 первых страницы, а остальные подгружать по мере "прохождения".

Дополнительно 2 - сохранять подгруженные данные чтобы не скачивать их повторно при заходе на старую страницу.

Дополнительно 3 - сохранять данные в сторадж чтобы не грузить их повторно и сохранять после перезагрузки страницы

Дополнительно 4 - в зависимости от гет-параметров, переключаться на нужную страницу
